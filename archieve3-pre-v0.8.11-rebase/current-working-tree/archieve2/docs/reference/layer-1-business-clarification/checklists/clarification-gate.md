# 业务澄清闸门清单

只有全部通过，才建议进入第二层。

## A. 目标是否清楚

- [ ] 已说明这个功能解决什么问题
- [ ] 已说明谁会使用它
- [ ] 已说明本次范围之外的内容

## B. 角色是否清楚

- [ ] 已识别关键角色
- [ ] 已说明每个角色的职责和限制
- [ ] 已识别是否存在管理员与普通使用者差异

## C. 流程是否闭环

- [ ] 主线流程完整
- [ ] 关键分支已补齐
- [ ] 异常流程已补齐
- [ ] 流程起点和终点明确

## D. 规则是否可执行

- [ ] 业务规则不是口号，而是可判断的规则
- [ ] 已说明规则优先级
- [ ] 已说明冲突时谁覆盖谁
- [ ] 已说明默认值和显式值的区别

## E. 状态与决策是否清楚

- [ ] 已列出关键状态
- [ ] 已列出状态切换触发条件
- [ ] 已用决策表表达复杂判断

## F. 风险是否暴露

- [ ] 已列出边界条件
- [ ] 已列出待确认问题
- [ ] 未决问题不会直接阻塞下一步工作，或已明确阻塞项

## G. 验收是否可落地

- [ ] 至少有 2 个可验证验收样例
- [ ] 验收样例覆盖正常路径
- [ ] 验收样例覆盖异常或边界路径

## 结论

- [ ] 允许进入第二层
- [ ] 暂不允许进入第二层，需继续澄清
