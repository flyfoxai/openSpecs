# 考勤请假申请

## Background

员工需要在统一入口提交请假申请，并在审批完成后看到清晰结果。

当前问题是请假规则、审批边界、页面入口和异常反馈分散，后续交付容易各自理解。

## Goals

- 建立员工提交请假申请的统一业务入口
- 建立主管审批、撤回、驳回的稳定主链路
- 为后续交付文档建立稳定的页面、用例、接口、表追踪基础

## In Scope

- 员工提交请假申请
- 员工查看申请列表与详情
- 主管审批或驳回申请
- 请假余额预占与审批结果通知
- 主链路验收样例

## Out Of Scope

- 排班引擎重算
- 薪资结算
- 移动端离线提交
- 多级会签审批

## Roles

- `ROLE-EMPLOYEE`：发起请假申请、查看列表、查看详情、撤回草稿或待审申请
- `ROLE-MANAGER`：审批或驳回本部门员工申请
- `ROLE-HR`：查看全局记录、处理例外规则和人工修正
- `ROLE-SYSTEM`：执行余额预占、通知、审计落账等副作用

## Success Criteria

- 员工能在单一申请页完成请假提交
- 主管能在审批页清晰看到审批依据并完成审批
- 页面、流程、规则、接口、表、验收样例之间建立稳定编号与回链
- 文档可以直接作为第二层交付设计输入

## Open Clarification Items

- 跨时区员工是否按申请人时区还是组织时区计算日期边界
- 半天请假是否允许和跨天申请同时出现
- 主管代理审批是否纳入当前范围
